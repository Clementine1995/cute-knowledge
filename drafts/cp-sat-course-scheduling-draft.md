- 变量直观：课程、老师、教室、时间槽都能画成可移动的小方块。
- 约束直观：老师不能同时上两节课，教室不能超员，实验课要跟在理论课后。
- 目标直观：尽量别让学生天天早八，别让老师来回跑太狠。
- 传播、冲突学习、剪枝、全局约束都能画出动作，不只是一堆术语。

## 5 页步骤版定稿

### 第 1 页：总览封面

- 页面目标：让读者 3 秒内知道这是 CP-SAT 算法科普，不只是排课插画。
- Step 标题：CP-SAT 是什么？
- 页面副标题：先用排课看懂它
- 每页一句话：先用排课来理解：它会边试、边排除、边记住走不通的原因。
- 必须出现的图中文字：CP-SAT、用排课理解、处理很多规则的安排算法、边试、边排除、记住死路。
- 画面描述：可爱的排课小助手站在课表前，旁边有一条迷你流程带：边试 -> 边排除 -> 记死路 -> 找方案。画面要先看到“CP-SAT 是什么？”，并明确排课只是这次用来理解它的例子。
- 承接关系：先知道它大概在干什么，下一页开始拆排课问题。

### 第 2 页：说清问题

- 页面目标：把排课问题说清楚，让 0 基础读者知道算法先看哪些信息。
- Step 标题：Step 1：先把排课问题写清楚
- 页面副标题：能安排什么？不能破什么？
- 每页一句话：先看哪些东西能安排、哪些规矩不能破、最后想让课表尽量好在哪里。
- 必须出现的图中文字：变量：课程/老师/教室/时间、约束：不能冲突、目标：尽量舒服。
- 画面描述：空白课表居中，左侧是课程、老师、教室、时间四种可移动小方块；右侧是“不能撞课”“教室够坐”等规则贴纸；底部是“少早八”“少跑腿”的目标小卡。
- 承接关系：排课问题写清后，下一页看一个安排会牵动哪些地方。

### 第 3 页：连锁影响

- 页面目标：解释 CP-SAT 不是瞎猜，一个决定会带出一串后果。
- Step 标题：Step 2：放一门课，看谁被影响
- 页面副标题：一个决定会牵动别的格子
- 每页一句话：比如高数放在周二上午，老师、教室、这个班的时间都会跟着被占住。
- 必须出现的图中文字：传播：连锁影响、放下高数、老师被占用、教室被锁住、班级不能撞课。
- 画面描述：小助手把“高数”放进周二上午，彩色连线连到老师、教室、班级三个卡片，卡片被轻轻锁住。阅读路径用箭头从课程块指向三个后果。
- 承接关系：如果这些后果推到最后不通，下一页讲它怎么记住原因。

### 第 4 页：冲突学习

- 页面目标：解释“记错题”不是拟人装饰，而是算法记住死路原因。
- Step 标题：Step 3：排不通，就记下原因
- 页面副标题：下次遇到，早点绕开
- 每页一句话：如果这几步安排把课表堵死了，它会记住是哪几个决定一起惹的事。
- 必须出现的图中文字：冲突：排不通、死路原因、这几步有问题、下次绕开。
- 画面描述：课表某处出现红色叉号和“此路不通”便签，小助手把原因写进错题本。错题本旁边用短箭头指向“下次绕开”。
- 承接关系：记住死路后，还要把明显没希望的路线早点剪掉。

### 第 5 页：总结流程

- 页面目标：做大体了解图，让读者带走 CP-SAT 的完整印象。
- Step 标题：Step 4：明显不行的安排，早点跳过
- 页面副标题：边试边排除，找靠谱方案
- 每页一句话：CP-SAT 不是把所有课表试完，而是边试边排除，最后找一个靠谱安排。
- 必须出现的图中文字：剪枝：早点跳过、多路一起找、靠谱方案、不是万能、边试边排除。
- 画面描述：用一条清晰流程线串起“写清问题 -> 放一门课 -> 看连锁影响 -> 记下原因 -> 跳过明显不行 -> 找方案”。旁边有剪刀剪掉灰色分支，几个小助手看不同路线，最后落到一张清爽课表。
- 承接关系：自然进入正文互动：你想让它帮你排什么？

## 小红书正文

### 推荐发布标题

CP-SAT 是什么？先用排课看懂它

### 正文

有没有觉得排课表这种事特别容易连环翻车？

你刚把高数塞进周二上午，体育老师突然冒出来：不行，我那会儿还有别的班。

然后教室也撞了。

班级也撞了。

课表直接开始打结。

今天讲的 CP-SAT，可以先把它想成一个很会排课的小助手。

但它厉害的地方，不是把所有课表都硬试一遍。那太累了，试到头发都白。

它会先看清楚几类东西：

课程、老师、教室、时间槽，这些是能被安排的变量。

老师不能同时上两节课，教室不能超员，这些是不能乱破的约束。

在不违反规则的前提下，尽量别让学生天天早八，别让老师跑断腿，这就是目标。

然后它开始试。

一旦把“高数”放进周二上午，它会立刻推出来：这个老师被占用了，这个班不能同时上别的课，这间教室也被锁住。

这叫传播。

不是瞎蒙，是一个决定带出一串后果。

如果推到后面发现死路，比如怎么排都满足不了某个班的课时，它也不会只是退一步重来。

它会记下来：刚才是哪几个决定把路堵死了。

有点像写错题本。下次看到相似局面，提前绕开。

路太多怎么办？

它还会先粗粗估一下。某条路看起来就会让学生连续四节专业课，那就别往下走了，早点剪掉。

遇到“实验课必须跟在理论课后”这种常见难题，也不用每次从零拆，可以用专门的处理方式。

任务再大一点，还可以几个搜索方向一起跑。一个方向踩过的坑，其他方向也能知道。

所以 CP-SAT 不是把所有可能都试完。

它是一边试，一边推理，一边排除没希望的路。

当然，这只是简化版理解。真实 CP-SAT 复杂得多，也不是万能工具。但如果你以后看到“排班、排课、排产、路线安排”这类问题，可以先想到它：哦，这是那种规则很多、选择很多、还想找个好方案的麻烦事。

你最想让这种小助手帮你排什么？课表、旅行路线，还是每天先做哪件事？

## 风险检查

- 已避免只讲“约束 + 目标”。
- 已覆盖传播、冲突学习、估算剪枝、全局约束、并行搜索。
- “估算剪枝”是通俗说法，真实实现涉及边界、松弛、启发式等，不等于随便估。
- “错题本”对应冲突学习/子句学习的比喻，不是字面记文本笔记。
- “专门的处理方式”对应全局约束和专用传播器，不代表所有问题都有现成模板。
- 已说明 CP-SAT 不是万能工具。

## 成图评审记录

- 最新 5 页版成图已经能让 0 基础读者大概理解：CP-SAT 不是专门排课的软件，而是适合处理规则多、选择多、还想找好方案的安排类问题。
- P1 需要保留一句定义感短句：处理很多规则的安排算法。否则读者容易只记得“排课例子”，不知道 CP-SAT 属于哪类方法。
- P2 到 P4 的步骤感成立：写清问题 -> 放一门课看连锁影响 -> 排不通就记下原因。
- P5 信息略多，但作为总结图可接受，底部“不是全试完，是边试边排除”建议保留。
- 推荐发布顺序：P1 总览封面 -> P2 Step 1 写清问题 -> P3 Step 2 看连锁影响 -> P4 Step 3 记下原因 -> P5 Step 4 跳过明显不行并总结。
- 剩余风险：P1 若不后期补定义句，算法定位仍会偏弱；P5 若生成过乱，优先单独重跑 P5，不必整套重跑。
