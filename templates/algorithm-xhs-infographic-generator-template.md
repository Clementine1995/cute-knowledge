# 小红书算法科普图文生成器模板

## 使用场景

当需要把一个算法、机制或 AI 概念转成小红书多图结构时，先用本模板定骨架，再进入文案定稿和 Image2 Prompt。

本模板固定的是“0 基础读者能看懂的解释结构”，不是固定某个生活例子。比如 CP-SAT 可以用排课、排班、排产或旅行安排解释；推荐算法可以用小店员、书架或奶茶口味解释；路径规划可以用地图、迷宫或通勤路线解释。

## 输入

- 算法名：
- 生活例子：
- 目标读者：0 基础小红书用户
- 页数：按算法步骤和难易程度动态决定，通常 3 到 6 页；复杂题拆上下篇
- 视觉风格：软萌治愈 + 贴纸手账
- 准确性边界：这是简化版理解，不写成严格定义

## 输出要求

必须输出：

- 每页作用
- 每页 Step 标题
- 每页副标题
- 每页一句话
- 必须出现的图中文字
- 画面结构
- 阅读路径
- 最后一页总结和互动问题
- 准确性风险提醒
- 0 基础文案自检
- 跨设备复现检查

## 固定解释骨架

### P1：这是什么

- 目标：3 秒内让读者知道正在看哪个算法或机制。
- 写法：直接出现算法名，不只写生活场景。
- 必须回答：它大概像什么？在这个生活例子里帮人做什么？

### P2：它解决什么麻烦

- 目标：从生活问题切入，让 0 基础读者知道为什么需要这个算法。
- 写法：先讲具体混乱，不从术语定义开头。
- 必须回答：如果不用它，人会卡在哪里？

### P3：它把问题拆成什么

- 目标：把复杂问题拆成几类看得见的对象。
- 写法：用 2 到 4 个短词，不堆术语。
- 常见拆法：变量、规则、目标、候选项、分数、路径、匹配对象。

### P4：它按什么步骤工作

- 目标：让读者知道从哪里看起，理解算法不是魔法。
- 写法：每页只讲一个动作，例如试一步、比较、推理、排除、记住、更新。
- 必须回答：这一步发生了什么？下一步为什么接得上？

### P5：一句话带走

- 目标：做总结图或大体了解图。
- 写法：用一条清晰流程线或一张最终结果图收住。
- 必须回答：读者看完应该记住哪句话？
- 硬要求：P5 不可省略。即使改成 4 页，也必须把最后一页做成总结图；不得只输出 2 页或 4 页解释图后直接结束。

## 生成提示词格式

```text
请根据以下输入，生成一套小红书算法科普图文结构。

算法名：[填写算法名]
生活例子：[填写生活例子]
目标读者：0 基础小红书用户
页数：请先按算法步骤和难易程度判断，通常 3 到 6 页
视觉风格：软萌治愈 + 贴纸手账

要求：
1. 不要把生活例子写死成通用模板；生活例子只服务这次算法解释。
2. P1 必须明确出现算法名。
3. 每页必须有 Step 标题、副标题、每页一句话、必须出现的图中文字、画面结构和阅读路径。
4. 每页只讲一个小步骤或一个关键变化。
5. 图中文字要短，优先 2 到 8 个字；长解释放在配文或后期排版。
6. 最后一页必须是总结图或大体了解图，不能省略。
7. 语气像朋友解释给朋友听，不用“首先/其次/最后”。
8. 保留准确性边界：这是简化版理解，不是严格技术定义。
9. 标题必须带明确对象和动作，不能写“先拆成三类东西”“开始处理”“进入下一步”这类缺少指代的句子。
10. 副标题必须补足读者会问的“拆什么、试什么、谁被影响、为什么这样做”。
11. 不要机械套用固定页数。先判断这个算法需要几步讲清楚，再定页数；早期 2 页/4 页对比稿是旧验证口径，不是当前默认生成结构。
```

## 0 基础文案自检

生成后必须逐页自检一次，发现问题先改文案再进入 Image2 Prompt。

| 检查项 | 说明 |
| --- | --- |
| 主语是否明确 | 读者能不能知道这页在讲哪个问题、哪一步、哪个对象。 |
| 动作是否具体 | 不写“处理、分析、拆解、优化”就结束，要写成“放一门课、看谁被影响、记下原因、早点跳过”。 |
| 副标题是否补疑问 | 标题如果会让读者问“把什么拆了？”“试什么？”“为什么？”，副标题必须回答。 |
| 术语是否有人话解释 | 可以出现传播、冲突、剪枝，但旁边必须跟“连锁影响、排不通、早点跳过”。 |
| 阅读路径是否明显 | 画面必须能看出从哪里开始读，以及下一步看哪里。 |
| 是否误把比喻当定义 | 生活例子只能写成“用它来理解”，不能写成算法本身就是这个例子。 |
| 单页是否过载 | 如果一页同时讲两个机制，拆页或删掉次要机制。 |

## 跨设备复现检查

如果在另一台电脑、另一个会话或外部 LLM 中重新生成，先逐项确认：

- [ ] 是否使用当前模板，而不是 README/PRD 里早期 2 页/4 页验证口径。
- [ ] 是否先根据算法步骤和难易程度判断页数，而不是机械套用 5 页或退回旧 2 页/4 页。
- [ ] 是否包含最后总结页或大体了解图。
- [ ] P1 是否明确算法名和一句定义感短句。
- [ ] 每页是否有 Step 标题、副标题、每页一句话和图中文字。
- [ ] 生活例子是否只是解释载体，没有被写成算法定义。

## 检查清单

- [ ] 算法名是否在第一页明确出现。
- [ ] 生活例子是否只是解释载体，没有被写成全项目固定场景。
- [ ] 每页是否有清楚的阅读起点。
- [ ] 每页标题是否明确写出“对象 + 动作”。
- [ ] 每页副标题是否解释了标题里可能悬空的词。
- [ ] 每页是否只讲一个步骤或关键变化。
- [ ] 术语旁边是否有人话解释。
- [ ] 图中文字是否足够短。
- [ ] 最后一页是否能让读者带走一句话。
- [ ] 是否避免退回旧的 2 页/4 页结构。
- [ ] 是否避免把简化比喻写成严格定义。
