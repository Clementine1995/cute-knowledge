# 小红书可爱图解知识账号需求文档 V1

## 1. 项目概述

本项目计划从小红书开始，打造一个用生动例子和可爱风图片解释复杂知识的内容账号。早期不做无边界泛科普大而全账号，而是先从身体机制、生活机制和 AI / 算法机制等高感知场景做验证。

目标是把普通人觉得遥远、抽象、难懂的概念，转化成生活化、故事化、视觉化的内容，让用户产生“原来是这样”的理解感，并愿意收藏、评论和转发。

## 2. 核心定位

### 账号定位

用可爱图解讲懂身体、生活和世界里的小机制。

### 内容关键词

- 可爱
- 治愈
- 通俗
- 有画面感
- 生活化
- 复杂知识简单讲
- 身体机制 / 生活机制 / AI / 算法 / 推荐 / 排序 / 平台机制

### 不建议的定位

- 不定位成纯算法科普账号或纯健康科普账号
- 不定位成硬核技术教学账号
- 不定位成考试型知识账号
- 不一开始做过宽、无边界的泛知识百科

## 3. 目标用户

### 主要用户

对世界运行机制感兴趣，但不想看太硬内容的普通小红书用户。

典型特征：

- 对身体、生活、AI、财经、商业、社会现象有好奇心
- 经常看到一些概念，但没有系统理解过
- 喜欢轻松、有画面感、可收藏的内容
- 不排斥知识内容，但不想被术语压住

### 次要用户

学生、职场新人、内容创作者、产品/运营/商业相关从业者。

他们可能更关注：

- 能不能快速理解一个概念
- 能不能拿去和别人解释
- 能不能启发选题、表达或工作思考

## 4. 已确认决策

- 早期渠道：只做小红书，不同步公众号、视频号、抖音
- 首批主题：身体机制、生活机制、AI / 算法机制
- 内容形态：纯图文
- 图片生成：使用 GPT-image2，由创作者自行生成图片
- 账号名称：暂未确定
- 文案立场：尽量中性解释，少做强观点输出
- 固定主角：暂不建立固定小动物、小机器人或拟人知识助手
- 内容长度：单页保持轻，整篇保持完整，页数根据复杂度决定，不为凑模板增加页面
- 视觉主风格：采用“软萌治愈 + 贴纸手账”，后续可按选题切换或扩展
- 默认画幅：小红书 3:4，后续可增加 4:5 或竖版长图
- 默认页数：短图文优先，不做长篇大论；根据机制步骤和难易程度动态定页数，简单题可 3 到 4 页，常规题 4 到 6 页，复杂题可拆上下篇；机制科普必须保留总结页
- 固定封面结构：暂不强制建立
- 账号名称和一句话简介：当前不作为优先事项
- 第一篇样稿：选择 CP-SAT
- 样稿版本：早期曾同时产出 2 页版和 4 页版做节奏对比；后续机制科普按步骤和难易程度动态定页数，并保留总结页
- 生产模板：暂不急着沉淀固定模板，先通过样稿验证
- CP-SAT 样稿场景：使用“安排座位”解释约束求解
- 样稿交付范围：默认输出标题、分镜、每页配文、画面描述、GPT-image2 Prompt 草案和总结页；如用户明确要求对比版本，再额外输出多版节奏方案

## 5. 用户需求

用户想要的不是完整课程，而是：

- 看懂一个经常听到但没弄明白的概念
- 通过生活例子理解抽象机制
- 用轻松方式获得一点“知识增量”
- 收藏以后可以回看或转发给别人
- 在评论区表达自己的经历和看法

## 6. MVP 内容范围

早期建议以“身体机制 + 生活机制 + AI / 算法机制”三条线做验证，优先从一个普通人有感的生活问题切入，再解释背后的一个核心机制。每篇只介绍一个核心机制，避免同时塞入太多医学、算法或平台概念。等基础内容跑通后，再过渡到“一个生活现象背后有多个机制”的复合选题。

### 系列一：身体里的小机制

主题：用温柔、生活化的方式解释身体运行里的常见机制，不做诊断或治疗建议。

候选选题：

- 为什么生理期不是每个月都准时？
- 为什么来姨妈前情绪会突然变敏感？
- 痛经为什么有的人轻、有的人很难熬？
- 为什么熬夜后更容易嘴馋或暴躁？
- 为什么压力大会影响睡眠和周期？
- 为什么有时候明明睡够了，还是觉得累？
- 为什么冷的时候身体会发抖？
- 为什么运动后第二天会酸？

### 系列二：生活里的算法

主题：用生活例子解释一个具体算法或机制，先让用户知道“这个算法大概在干什么”。

候选选题：

- 什么是推荐算法？像一个记性很好的小店员
- 什么是排序算法？像排队时决定谁先到前面
- 什么是路径规划？像地图帮你在路口做选择
- 什么是风控算法？像支付前的小安检
- 什么是匹配算法？像把订单和司机配成一对
- 什么是预测模型？像根据过去天气猜明天会不会下雨
- 什么是多臂老虎机算法？像一边试奶茶一边找到最好喝的口味
- 什么是聚类？像把相似口味的人分到一桌
- 什么是异常检测？像在人群里发现一个特别不一样的动作

### 系列三：AI 小助手背后的机制

主题：解释普通人正在接触的 AI 工具、智能助手和自动化功能背后的基本原理。

候选选题：

- AI 为什么能猜你下一句话？
- AI 总结文章时，到底是在“理解”还是在“压缩”？
- 为什么 AI 有时候会一本正经地胡说？
- AI 画图为什么同一句提示词每次都不一样？
- 为什么 AI 能模仿某种画风或语气？
- AI 助手为什么需要“提示词”？
- 为什么有些 AI 回答看起来聪明，但细问就露馅？

### 系列四：平台小机关

主题：作为后续进阶系列，解释平台产品中常见的推荐、风控、定价、分配和排序机制。这个系列通常会涉及多个算法，适合在基础概念内容跑通后再做。

候选选题：

- 为什么打车越急越贵？
- 为什么平台会给不同人发不同优惠券？
- 为什么你搜到的第一条不一定是最好的？
- 为什么短视频越刷越像为你定制？
- 为什么平台知道你可能快要流失了？
- 为什么新用户常常比老用户优惠更多？
- 为什么同一个商品，不同时间看到的价格可能不一样？

### 后续扩展方向：可爱财经课

主题：解释普通人常听到但容易模糊的财经概念。

候选选题：

- 通胀到底是什么意思？为什么钱会变“不耐花”？
- 利率为什么像“钱的租金”？
- 降息为什么会影响房贷、股市和消费？
- 汇率为什么会变来变去？
- 债券到底是不是一种“可转卖的借条”？
- 做空为什么不是简单地“看跌”？
- 为什么银行不是把你的钱原封不动放在保险柜里？

### 后续扩展方向：历史上的系统故障

主题：用历史事件解释泡沫、信心、杠杆、资源和制度如何影响社会。

候选选题：

- 郁金香泡沫：当大家都相信还能卖给下一个人
- 1929 大萧条：信心、杠杆和挤兑如何连锁反应
- 南海泡沫：一个故事如何比真实资产跑得更快
- 石油危机：一种资源如何影响全世界价格
- 布雷顿森林体系：全世界曾经如何约定钱的规则

## 7. 内容形态

### 单篇笔记结构

建议每篇采用稳定结构：

1. 生活问题开头：用用户熟悉的场景切入
2. 可爱比喻解释：把抽象概念转成角色、故事或小剧场
3. 真实机制拆解：点出背后的概念，但避免术语堆叠
4. 一句话总结：给用户一个能带走的理解
5. 互动问题：引导评论区分享经历或看法

### 多图叙事要求

部分身体、生活、算法或平台机制无法用一张图讲清楚，需要通过多图连续解释。多图内容必须保持视觉和叙事连贯，避免每一页像独立海报。

要求：

- 每篇围绕一个核心问题展开，不在同一篇里塞太多概念
- 每一页只讲一个小步骤或一个关键变化
- 固定主角、场景、道具和色彩风格，保证生图结果一致
- 角色状态要随着解释推进发生变化，例如疑惑、尝试、发现、理解
- 前后页之间要有明确承接关系，例如“刚才发生了什么”“下一步系统怎么判断”
- 最后一页必须收束成一句可记住的总结

建议多图结构：

1. 封面：提出生活问题
2. 场景：展示用户熟悉的日常情境
3. 第一步机制：身体、系统或场景先发生什么
4. 第二步机制：后面如何变化、比较或判断
5. 总结：用一句话讲清楚背后的机制逻辑

页数根据复杂度决定。能 2 页讲清楚就不要扩成 3 页；常规题控制在 3 到 5 页；复杂概念优先拆成上下篇，而不是一篇放太多页。

### 生活化解释原则

科普时优先使用生活中常见、具体、可想象的例子，不从算法名或技术定义开始。

要求：

- 用身体感受、外卖、打车、购物、导航、排队、便利店、客服、短视频等日常场景切入
- 先讲“你遇到过什么”，再讲“背后是什么机制”
- 把机制动作翻译成人类能感知的动作，例如“变多”“变少”“提醒”“比较”“排队”“试探”“打分”“分配”“绕路”
- 避免连续出现多个技术词；必要术语放在后半段轻轻点出
- 不为了可爱而牺牲准确性，复杂处可以说明“这是简化版理解”
- 身体健康类内容只讲常识机制和个体差异，不做诊断、治疗建议或“你一定是某种情况”的判断。

示例表达方向：

- 不说“推荐系统基于用户画像和协同过滤进行个性化排序”
- 改成“平台像一个记性很好的小店员，它记得你常看什么，也会看看和你口味相似的人喜欢什么”

### 图片风格

建议采用可爱、轻量、统一角色体系。

早期可以在以下视觉方向中选择一种主风格，不建议同时混用太多：

- 软萌治愈：温暖、圆润、低压力，适合“可爱解释复杂知识”的主定位
- 贴纸手账：轻松、生活化、适合小红书收藏和多图翻页
- 扁平插画：清晰、稳定、适合讲机制和流程
- 绘本风：故事感强，但制作成本和一致性要求更高
- Q 版角色：记忆点强，但如果不建立固定主角，暂时不作为主方向

当前建议优先选择：

- 主风格：软萌治愈
- 辅助形式：贴纸手账

这样既能保持可爱感，也方便用多图解释复杂机制。

可复用元素：

- 普通人角色：代表用户，可以每篇变化，不设固定主角
- 小机器人元素：代表算法或系统，可以作为图形符号，不必成为固定角色
- 小钱袋元素：代表资金、价格、成本
- 小地图元素：代表路径、调度、选择
- 小喇叭元素：代表平台信号或规则
- 小天平元素：代表权衡、排序、公平性

### 图片形式

早期优先做纯图文 carousel：

- 1 张封面图
- 1 到 3 张解释图
- 1 张总结图

图片由创作者使用 GPT-image2 自行生图，文案和通俗化比喻可结合 LLM 辅助生成与打磨。不建议一开始做太重的视频动画、真人出镜或复杂动画，除非后续数据证明图文已经跑通。

## 8. 内容语气

### 应该像

- 朋友解释给朋友听
- 有一点故事感
- 轻松但不糊弄
- 简单但不幼稚
- 有趣但不标题党
- 中性解释，不刻意制造立场冲突

### 避免

- 上来解释术语定义
- 大段历史背景
- 过度学术化
- 把复杂问题讲成唯一答案
- 只追求可爱但没有知识增量
- 过度批判平台或算法
- 用情绪化判断替代机制解释

## 9. 首批验证计划

### 建议周期

先做 4 周验证。

### 发文频率

每周 3 到 5 篇。

### 首批数量

建议先完成 12 到 20 篇内容，用来判断哪些系列和表达方式更有潜力。

### 选题比例

早期建议：

- 身体机制：40%
- 生活和平台机制：30%
- AI / 算法机制：30%

如果某个系列明显跑出数据，再加大比例。

### 选题来源

后期选题可以从四类来源持续产生：

1. 身体机制清单：生理期、睡眠、情绪、疼痛、疲劳、压力、饮食、运动恢复等
2. 算法概念清单：推荐算法、排序算法、路径规划、匹配算法、风控算法、预测模型、聚类、异常检测、Bandit、A/B 测试等
3. 日常生活场景：外卖、打车、购物、导航、客服、支付、短视频、招聘、快递、便利店
4. 用户评论反馈：从评论区收集“这个能不能讲讲”“为什么会这样”的问题
5. 热点和新工具：当 AI 产品、平台机制、健康生活或社会话题变热时，用中性方式解释背后的基础机制

选题优先级规则：

- 优先选一个概念能讲清楚的题
- 优先选生活中有对应场景的题
- 优先选 4 到 5 页能讲完的题
- 暂时避开需要大量背景知识或争议很强的题
- 复杂平台现象可以拆成多个基础算法先讲，再做合集或进阶篇
- 身体健康类选题必须能用普通科普讲清楚，并保留“个体差异”和“异常情况请看医生”的边界。

## 10. 成功指标

### 核心指标

- 收藏率
- 评论率
- 转发率
- 完读率或多图滑动完成度

### 观察指标

- 哪类封面标题点击更高
- 哪类比喻更容易被评论复述
- 用户是否在评论区说“终于懂了”
- 用户是否主动点题或催更
- 是否出现跨平台转发需求

### 阶段目标

V1 阶段不以粉丝数作为唯一目标，而是验证：

- 内容方向是否有人愿意看
- 可爱图解风格是否被记住
- 用户是否愿意收藏
- 是否能形成稳定选题生产机制

## 11. 非目标范围

早期不做：

- 系统课程
- 硬核算法教学
- 直播课或知识付费
- 全领域百科式科普
- 复杂视频动画
- 真人出镜
- 多平台同步运营
- 高成本原创 IP 世界观
- 固定主角 IP 设定
- 每篇都追求完整学术严谨

这些可以等账号找到稳定流量后再考虑。

## 12. 工作流程

### 单篇内容生产流程

1. 选题：选择一个生活问题或常见概念
2. 拆解：找到背后的 1 到 3 个核心机制
3. 比喻：设计一个可爱、直观的小故事
4. 分镜：根据复杂度拆成 2 到 5 页连续图文，明确每页画面、角色、道具和承接关系
5. 提示词：为每页生成图片提示词，并保持角色、场景、风格一致
6. 作图：用统一角色和视觉模板出图
7. 发布：配标题、正文、话题标签
8. 复盘：记录数据和评论反馈

### 复盘维度

- 标题是否吸引点击
- 封面是否一眼看懂
- 比喻是否帮助理解
- 哪一页最容易被截图或评论
- 评论区有没有提出下一篇选题

## 13. 风险与应对

### 风险一：内容太泛，用户记不住账号

应对：早期只保留 2 到 3 个系列，用统一视觉和统一表达方式建立记忆点。

### 风险二：可爱风掩盖知识价值

应对：每篇必须有一个清晰的“带走点”，让用户看完真的理解一个机制。

### 风险三：概念解释不严谨

应对：避免绝对化表达，多用“通常”“可以理解为”“简化来说”，必要时在正文补充边界。

### 风险四：健康内容被误读成医疗建议

应对：身体机制内容只做普通科普，必须说明个体差异；涉及严重痛经、长期紊乱、大量出血、持续不适等情况，要提醒寻求专业医生帮助。

### 风险五：选题太硬，点击不足

应对：标题从生活问题出发，不从术语出发。例如讲“推荐算法”时，标题写“为什么你刚看完一个东西，APP 就开始推荐？”；讲生理期时，标题写“为什么生理期不是每个月都准时？”

### 风险六：制作成本过高，难以持续

应对：早期使用固定角色、固定版式、固定篇幅，不追求每篇重新设计。

## 14. 可引入的 Skill 分工

后续可以引入小红书相关 Skill，但不建议直接交给某一个 Skill 全自动完成。由于图片由创作者使用图像模型自行生成，Skill 不负责实际生图，只负责把内容拆成可执行的图文方案：选题与笔记策划、连续分镜、画面描述、角色一致性要求、最终发布文案。

### 推荐主流程

1. 文案策划：使用 `xhs-notes-skill` 类 Skill 生成标题、正文、标签、评论钩子和风险检查
2. 分镜策划：使用 `xhs-images` 类 Skill 拆解多图结构，并为每页生成画面说明、构图建议和生图 Prompt 草案
3. 视觉模板：可少量参考 `blcaptain-style-skill` 的组图排版方法，但不要完全照搬其偏杂志/证据卡风格
4. 发布复盘：发布后用笔记复盘能力分析封面、标题、收藏率、评论率和下一篇实验方向

## 15. 后期需求：自动热点收集与选题池

该需求不进入 MVP，等账号完成基础内容验证后再考虑。

### 目标

建立一个半自动选题来源机制，定期收集身体机制、生活机制、AI、算法、平台机制、小红书内容趋势和用户评论反馈，筛选出适合本账号的图文选题。

### 适用阶段

- 已经发布 10 到 20 篇基础内容
- 已经初步确认用户更喜欢哪些身体、生活或算法主题
- 需要提升选题效率，减少每次从零想题

### 可借鉴的 Skill 经验

后期热点收集不应只依赖一个 Skill，可以从前面候选的小红书相关 Skill 中拆能力使用。

| Skill | 可借鉴能力 | 在本项目里的位置 |
| --- | --- | --- |
| `xhs-notes-skill` | 联网刷新同类主题、近期热点、平台表达、热门标签；生成标题、正文、标签、评论钩子；发布后复盘 | 热点收集和选题池主参考 |
| `xhs-writer-skill` | 小红书爆款标题、正文结构、基于 GPT-image-2 的图文生成流程 | 标题备选和表达包装参考，不直接决定选题 |
| `rednote-skills` | 小红书 / RedNote 相关工作流集合 | 待确认具体能力后，可补充到热点采集或发布流程 |
| `blcaptain-style-skill` | 小红书图文卡片、视觉语言、排版稳定性、跨画幅输出 | 不负责热点，适合后期视觉模板和卡片排版参考 |
| `xhs-images` 类 Skill | 多图拆解、风格选择、布局选择、每页画面 Prompt 草案 | 不负责热点来源，负责把已筛选选题变成多图分镜 |

当前优先参考：

- 热点和趋势：`xhs-notes-skill`
- 标题表达：`xhs-notes-skill` + `xhs-writer-skill`
- 分镜提示词：`xhs-images`
- 视觉排版：`blcaptain-style-skill`
- 其他 RedNote 流程：`rednote-skills` 待确认后再纳入

### 选题来源

- 身体和健康生活热点：生理期、睡眠、情绪、疼痛、疲劳、压力等高频困惑
- AI 新闻和工具热点：新模型、新功能、新产品、新争议
- 平台机制热点：推荐、搜索、风控、派单、定价、流量分发
- 小红书站内趋势：相关关键词、热门标题表达、常见评论问题
- 用户评论反馈：用户问“为什么会这样”“能不能讲这个”
- 过往笔记复盘：收藏率高、评论多、完读好的主题延展

### 筛选规则

自动收集到的热点不能直接变成选题，需要经过筛选：

- 是否能拆成一个简单机制
- 是否能用生活例子解释
- 是否适合 4 到 5 页图文讲完
- 是否适合中性解释，避免强争议表达
- 是否符合“软萌治愈 + 贴纸手账”的表达方式

### 输出格式

每次热点收集后，输出一个候选选题池：

| 字段 | 说明 |
| --- | --- |
| 热点来源 | 来自 AI 新闻、平台机制、小红书趋势、评论反馈或复盘 |
| 原始热点 | 收集到的原始事件、词条或用户问题 |
| 可解释概念 | 可转化成哪个身体、生活、算法或平台机制 |
| 推荐选题 | 可直接进入脚本生产的标题方向 |
| 难度 | 简单 / 中等 / 偏复杂 |
| 推荐优先级 | 高 / 中 / 低 |
| 备注 | 是否需要拆成上下篇，是否有时效风险 |

### 示例

| 原始热点 | 可解释概念 | 推荐选题 |
| --- | --- | --- |
| 某平台推荐机制引发讨论 | 推荐算法 / 排序 | 什么是推荐算法？像一个记性很好的小店员 |
| AI 工具回答错误被吐槽 | 幻觉 / 概率生成 | AI 为什么会一本正经地胡说？ |
| 打车高峰期价格波动 | 动态定价 / 供需匹配 | 什么是动态定价？像城市里的小喇叭 |
| 生理期总是不准时 | 月经周期 / 激素波动 / 个体差异 | 为什么生理期不是每个月都准时？ |
